OwnerRez OAuth Connection
GeoWaypoint connects to OwnerRez via OAuth 2.0 to sync availability and route bookings
Connected to OwnerRez
Account: Lakeside RV Resort LLC  ·  Authorized Apr 2, 2026  ·  Expires Jan 2027
Auth Flow
OAuth 2 Authorization Code Grant (RFC 6749-Section 4.1). No PKCE — server-side client secret exchange. No scopes required; access is granted at the app level by the OwnerRez user.
Grant type: authorization_code
Base URL: app.ownerrez.com/api/v2/
Auth header: Authorization: bearer at_…
Required: User-Agent: GeoWaypoint/1.0 (c_{clientId})
Scopes: none (not used by OwnerRez)
Callback: api.geowaypoint.io/oauth/ownerrez/callback
Availability Updates — Webhooks
OwnerRez pushes real-time booking events to GeoWaypoint via webhook — no polling. GeoWaypoint updates site status (available/occupied) immediately when OwnerRez fires a booking create, change, or cancel event.
GeoWaypoint Webhook Receiver
api.geowaypoint.io/webhooks/ownerrez
Enter this URL in your OwnerRez OAuth App under Developer/API Settings → Webhooks, along with a Username + Password for basic auth verification.

Booking Flow
When a guest clicks "Book This Site," GeoWaypoint creates a Quote via the OwnerRez API, receives a hosted PaymentForm URL, then redirects the guest there to complete checkout. GeoWaypoint never processes payment directly.
1Guest clicks "Book This Site"
2GeoWaypoint POSTs to app.ownerrez.com/api/quotes
3OR returns { PaymentForm: "https://…" }
4Guest redirected to OwnerRez checkout
Rate Display
Pulls nightly rates from OwnerRez via the properties API. Requires "WordPress plugin + Integrated websites" to be enabled on the OwnerRez account first.
Show OwnerRez rates in popup
Prerequisite: enable WordPress plugin + Integrated websites in your OwnerRez account before rates will pull through.
For developers (Cursor): Register your OAuth App at Settings → Advanced Tools → Developer/API Settings in OwnerRez. Set the OAuth Redirect URL to https://api.geowaypoint.io/oauth/ownerrez/callback. Every API call to app.ownerrez.com/api/v2/* requires both the bearer token header and User-Agent: GeoWaypoint/1.0 (c_{clientId}). Webhooks require Basic Auth set on the OwnerRez app config, not OAuth. See OwnerRez OAuth App docs ↗

Connect OwnerRez
4 quick steps · takes about 2 minutes
1
Log into OwnerRez
Make sure you're signed in to the OwnerRez account for your resort at app.ownerrez.com
2
Click "Authorize GeoWaypoint" below
You'll be redirected to OwnerRez. They'll show you exactly what GeoWaypoint is requesting access to — availability data, property names, and the ability to create quotes on your behalf.
properties.read bookings.read quotes.write
These are the only permissions GeoWaypoint requests. We never read payment info or modify existing bookings.
3
Approve on OwnerRez
OwnerRez will show a permission screen. Click "Allow" and you'll be redirected back to GeoWaypoint automatically.
4
Map your properties
Once connected, you'll link each GeoWaypoint site marker to its matching OwnerRez property. This takes about 5 minutes and only needs to be done once.
Security: GeoWaypoint uses OwnerRez's official OAuth 2.0 API. Your OwnerRez password is never shared with GeoWaypoint. You can revoke access from OwnerRez Settings → API Apps at any time.
